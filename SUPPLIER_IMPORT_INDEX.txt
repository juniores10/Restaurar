CSV COLUMN MAPPING:
================================================================================

CSV Column          → Database Column    Required?  Notes
─────────────────────────────────────────────────────────────
CNPJ               → cnpj               No         Tax ID or CPF
CLIENTE            → name               Yes        Company name
MUNICÍPIO          → city               Yes        City name
UF                 → state              Yes        2-letter code
CEP                → zip_code           No         Postal code

Alternate column names recognized:
  - CLIENTE, NOME, NAME, RAZAO SOCIAL, RAZSOCIAL → name
  - MUNICÍPIO, MUNICIPIO, CIDADE, CITY → city
  - UF, STATE, ESTADO → state
  - CEP, ZIP, ZIPCODE → zip_code
  - CNPJ, CPF → cnpj

================================================================================
SQL GENERATION DETAILS:
================================================================================

Temp Table: csv_suppliers
├─ cnpj (text)
├─ name (text)
├─ city (text)
├─ state (text)
└─ zip_code (text)
   source_row (integer) - for debugging

Step 1: CREATE TEMP TABLE csv_suppliers
Step 2: INSERT INTO csv_suppliers (FROM CSV data)
Step 3: UPDATE suppliers WHERE cnpj IS NULL (matching by location + name)
Step 4: INSERT INTO suppliers WHERE NOT EXISTS (new records)
Step 5: DROP TABLE csv_suppliers

Matching Logic:
├─ UPDATE: Matches city + state + (exact name OR partial name OR trade_name)
└─ INSERT: Checks CNPJ or (name + city + state) combinations

================================================================================
ERROR HANDLING:
================================================================================

Warnings (non-fatal):
- Missing required fields (name, city, state)
  → Row is skipped with warning message
  → Processing continues with remaining rows

Errors (fatal):
- CSV file not found
  → Creates sample SQL with template data
- Empty CSV
  → Error message, exits with code 1
- Malformed CSV
  → Attempts to parse, skips invalid rows

Output:
- All warnings and info logged to console
- SQL file still generated even with warnings
- Check console output for details

TROUBLESHOOTING MATRIX:

"Duplicate records created"    → Verify CSV has unique values
